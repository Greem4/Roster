# Лекарства: фото → JSON → БД

**Живая таблица `medicines` — единственный источник правды.** Удаляешь позиции в приложении или SQL — они не вернутся сами. Новые накладные — только через `rx_import_medicines` ниже.

**Не запускай** `./scripts/internal/import.sh` и `import-medicines-invoices.py` — они читают архивный `medicines-invoices.json` и могут **вернуть уже удалённые** строки (а `import.sh` ещё и очищает всю таблицу).

**Ошибка `function rx_import_medicines does not exist`:** один раз выполни **`rx-import-install.sql`** в этой папке (или `docker compose exec api alembic upgrade head` после деплоя API с миграцией **015**).

## Обычный способ — одна команда в консоли SQL

Открой БД (`roster-db-auto` в IntelliJ или `./scripts/internal/tunnel-db.sh` + psql), вставь JSON в шаблон и выполни:

```sql
SELECT rx_import_medicines('[
  {"name": "Глицин таб. 100мг №100", "series": "480723", "expiry": "01/06/26"},
  {"name": "Новокаин 5мг/мл 10мл №10", "series": "470623", "expiry": "01/06/26"}
]'::jsonb);
```

Ответ: `Добавлено: 2, всего в medicines: 87`.

- **Только добавляет** новые строки; существующие не меняет и не удаляет.
- Дубликат (то же name + series + expiry) — пропуск.
- Старый JSON в репозитории **не используется**.

Готовый файл для правки: **`rx-import.sql`** в этом каталоге.

### Поля

| Поле | Пример |
|------|--------|
| `name` | `"Фуросемид р-р для в/в и в/м введ. 2мл №5"` |
| `series` | `"6/820524"` |
| `expiry` | `"01/05/26"` — строго **DD/MM/YY** |

Срок **30/31** на упаковке → в JSON пиши **01** того же месяца (так же обработает и функция).

## Промпт для ИИ (фото накладной)

Прикрепи фото и вставь:

```
Из приложенных фото накладной/упаковок извлеки список лекарств.

Верни ТОЛЬКО JSON-массив объектов без markdown. Каждый объект:
- "name" — полное наименование
- "series" — серия (партия)
- "expiry" — срок DD/MM/YY (день и месяц с ведущим нулём)

Правила: одна позиция на серию; 30/31 → 01 того же месяца; не выдумывай то, чего нет на фото.

Пример: {"name": "Глицин таб. 100мг №100", "series": "480723", "expiry": "01/06/26"}
```

JSON из ответа ИИ — сразу в `SELECT rx_import_medicines('…'::jsonb)`.

## Архив (не для повседневной работы)

`medicines-invoices.json` — снимок первой заливки. **Не импортировать** после того как ведёшь список в БД вручную: скрипты из этого файла подмешают или восстановят старые позиции, которые ты уже удалил.
